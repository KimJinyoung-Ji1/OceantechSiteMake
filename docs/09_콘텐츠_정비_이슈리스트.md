# 09. 콘텐츠 정비 이슈리스트

## 즉시 정비 필요
### 1. 깨진 한글 데이터
- 현재 `src/lib/constants.ts`, `src/lib/i18n/ko.json` 등에 한글 깨짐이 존재한다.
- 이 상태로는 디자인 수정만으로 해결되지 않는다.
- 1차 구현 전에 한국어 원문 데이터를 다시 입력해야 한다.

### 2. 영문 데이터 체계 부족
- 한/영 다국어 구조는 있으나 확장 섹션 기준으로 영문 카피 재정비가 필요하다.
- 신규 섹션이 늘어나므로 `en.json`도 같이 재구성해야 한다.

### 3. 실자료 연결 구조 부족
- 기술력/인증현황 페이지에서 실문서 노출 체계가 아직 약하다.
- 문서 미리보기, 상세 보기, 파일 링크 방식을 명확히 통일해야 한다.

### 4. 페이지 밀도 부족
- 홈과 서브페이지 모두 자료 양에 비해 섹션 구성이 비어 보인다.
- 실문서, 뉴스, 성과, 신뢰 지표를 더 구조적으로 배치해야 한다.

## 구현 시 주의
### 1. 데이터 소스
- 확정 연락처/주소/도메인은 `코드전달용_확정정보.md` 기준으로 통일
- 교체 가능 정보는 상수 한 곳에서 관리

### 2. 디자인
- 차콜/샌드/베이지 금지
- 바다톤 4색 유지
- 라이트 모드만 지원

### 3. 자산 반영
- 인증서/시험성적서/수상 자료는 누락 없이 반영
- 녹색인증 마크는 실제 PNG 사용
- 바다/해양 사진은 장식이 아니라 섹션 구분 리듬용으로 사용

### 4. 개발 방식
- 공통 레이아웃 먼저
- 데이터 구조 다음
- 페이지별 섹션 리디자인 순서
- 마지막에 빌드 검증
