# 주요업무계획 수집기 실수와 개선 원칙

## 1. 이번 작업에서 확인된 실수

- 검색 결과에서 `주요업무계획` 키워드만 보고 바로 저장했다.
- `목록 페이지`, `상세 페이지`, `첨부 원문`, `e-book`을 구분하지 못했다.
- `사진`, `보도자료`, `회의록`, `인증서`, `재정공시`, `조례`, `검색결과`를 원문으로 오인했다.
- `onclick="fn_egov_downFile(...)"` 같은 자바스크립트 기반 첨부를 추적하지 못했다.
- `e-book`은 전체 프린트 PDF로 받아야 하는데, 대표 이미지나 첫 화면만 저장한 경우가 있었다.
- 페이지 캡처 PDF를 원문 대체로 허용한 것이 가장 큰 오판이었다.

## 2. 앞으로 지켜야 할 강제 규칙

- `검색 -> 바로 저장` 금지
- 먼저 아래 5가지 중 어떤 페이지인지 분류
  - 검색 결과
  - 목록 페이지
  - 게시글 상세
  - 직접 파일 링크
  - e-book 뷰어
- `목록 페이지`면 반드시 `2026 + 주요업무/업무계획/추진계획/시행계획` 제목의 상세로 한 단계 더 내려갈 것
- `게시글 상세`에서는 첨부 추출 방식을 모두 검사
  - 일반 `href` 다운로드 링크
  - `downloadBbsFile.do`, `FileDown.do` 계열
  - `onclick fn_egov_downFile(...)` 계열
- `e-book`이면 반드시 전체 페이지 기준으로 PDF 생성
- 원문 첨부를 찾지 못하면 `페이지 캡처 저장 금지`, 실패로 기록할 것

## 3. 저장 전 검증 규칙

- 1차 검증
  - 파일 형식이 실제 PDF/HWP/HWPX인지 확인
  - HTML 응답, 이미지 단일파일, 인증서, 사진은 제외
- 2차 검증
  - 첫 페이지 또는 앞부분 텍스트에 아래 키워드가 있는지 확인
    - `2026`
    - `주요업무`
    - `업무계획`
    - `추진계획`
    - `시행계획`
- 3차 검증
  - 아래 제외 키워드가 있으면 오검출 후보로 강등
    - `보도자료`
    - `사진`
    - `회의록`
    - `인증서`
    - `재정공시`
    - `조례`
    - `시정뉴스`
    - `검색결과`
- 4차 검증
  - 사용자 확인 전까지는 최종본으로 취급하지 말 것
  - 사용자 확인 합격본은 `파일명 앞에 1` 규칙을 기준으로 삼을 것

## 4. 대표 유형별 학습

### 하남시

- 검색으로 바로 상세가 아니라 `시정 주요업무계획 목록`이 먼저 잡혔다.
- 실제로는 목록에서 `2026년 하남시 시정 주요업무계획` 게시글로 다시 들어가야 했다.
- 교훈: 목록형 게시판은 제목 링크를 한 단계 더 타야 한다.

### 안성시

- 페이지는 맞게 들어갔지만 첨부가 `onclick fn_egov_downFile(atchFileId, fileSn)` 구조였다.
- 단순 `href` 수집만으로는 절대 잡을 수 없다.
- 교훈: 자바스크립트 다운로드 함수 파싱이 필수다.

### 강릉시

- e-book형 문서는 대표 이미지나 첫 화면 저장으로는 안 된다.
- 전체 페이지 SVG/뷰어 기준으로 프린트 PDF를 만들어야 했다.
- 교훈: e-book은 별도 수집 모드가 필요하다.

### 시흥시 / 횡성군 / 홍천군

- 직접 다운로드 PDF를 잡으면 원문 확보가 가능했다.
- 교훈: 원문 직접 링크가 보이면 페이지 캡처보다 첨부 원문을 항상 우선한다.

## 5. 운영 원칙

- 잘못 받은 파일을 많이 만드는 것보다, 적게 받아도 정확한 파일만 저장하는 것이 우선이다.
- `불확실하면 저장하지 말고 실패 처리`가 맞다.
- 사용자 확인 결과를 가장 강한 정답 데이터로 사용한다.
- 오검출 사례는 새 규칙으로 즉시 반영한다.
