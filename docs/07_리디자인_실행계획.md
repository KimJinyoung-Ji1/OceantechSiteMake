# 07. 리디자인 실행계획

## 목적
- 기존 6개 페이지 구조는 유지하되, 전체 디자인 퀄리티를 최신 수준으로 상향한다.
- 전달된 실자료를 빠짐없이 페이지 구조 안에 체계적으로 재배치한다.
- 구현 전 문서 기준을 먼저 확정하고, 승인 후 로컬에서만 코드 작업한다.

## 전제 조건
- 작업 기준 문서: `docs/01~06`
- 추가 지시 기준: `클로드코드전달정보/디자인수정_지시서.md`, `코드전달용_확정정보.md`
- 작업 범위: 로컬 수정만 진행
- 금지: 원격 push, 배포, Vercel 조작

## 핵심 방향
### 1. 디자인
- 컬러는 바다톤 4색만 유지한다.
  - `#021097`
  - `#0168EF`
  - `#03E9F8`
  - `#17E9B5`
- 차콜, 샌드, 베이지 계열은 제거한다.
- 전체 텍스트 크기를 전반적으로 상향한다.
- 섹션 폭을 넓혀 화면을 더 꽉 채운다.
- 서브페이지의 핵심 헤드라인과 주요 블록은 중앙 정렬로 재구성한다.
- 섹션 사이마다 실제 바다/항구/어업 현장 사진을 리듬감 있게 삽입한다.

### 2. 정보 구조
- 홈은 브랜드 소개와 인증 신뢰 형성에 집중한다.
- 회사소개는 대표 메시지, 비전, 연혁, 회사 개요를 신뢰형 정보 구조로 정리한다.
- 기술력은 실제 등록증/시험성적서 뷰어 중심으로 재설계한다.
- 인증현황은 실문서 갤러리 중심으로 재설계한다.
- 친환경은 납추 vs 아연추 비교, SVHC 235종 불검출, 환경 효과를 강하게 보여준다.
- 문의하기는 주소/연락처/카카오맵을 확정 정보 기준으로 재정렬한다.

## 현재 코드 기준 주요 문제
- `src/lib/constants.ts`, `src/lib/i18n/ko.json` 등 텍스트 소스가 깨져 있어 콘텐츠 소스부터 재작성해야 한다.
- 일부 홈 디자인 수정은 반영돼 있으나, 전달 자료 기준으로는 아직 정보 밀도와 완성도가 부족하다.
- 외부/실문서 자산을 페이지별로 충분히 구조화하지 못했다.
- 기술력/인증현황 페이지가 “실제 자료를 체계적으로 보여주는 페이지” 수준까지 올라오지 않았다.

## 페이지별 목표
### 홈
- 수협식 2단 헤더
- 풀폭 히어로
- 우측 정사각형 녹색인증 카드
- 녹색인증 스트립
- 신뢰 지표 섹션
- 자동 슬라이드 뉴스 배너
- 인증 요약 카드
- CTA, 푸터 재정렬

### 회사소개
- 서브 히어로를 더 고급스럽게 재구성
- 대표 인사말 가독성 상향
- 비전/미션/핵심가치 카드 리디자인
- 연혁 타임라인 리디자인
- 회사 개요 정보 카드 정리

### 기술력
- 특허/시험성적서/기술 검증 자료를 탭형 또는 카테고리형으로 재구성
- 각 자료에 실제 문서 보기 동작 연결
- 카드, 모달, 미리보기 이미지 전면 정리

### 인증현황
- 녹색기술인증과 녹색기술제품확인서를 최상단 대표 콘텐츠로 강조
- 나머지 인증/수상/등록/사업자 자료를 동일 비율 카드 시스템으로 통일
- 실제 이미지 또는 PDF 열람 연결

### 친환경
- 비교표, 실험 결과, SVHC 결과, 현장 적용 성과를 스토리 흐름으로 재배치
- 숫자/인포그래픽/실문서 근거를 결합

### 문의하기
- 확정 연락처 반영
- 메인 주소 기준 카카오맵 정렬
- 정보 카드와 폼 레이아웃 개선

## 자료 반영 원칙
- 실제 파일이 있으면 우선 반영한다.
- 확정 정보와 임시 정보는 문서와 코드에서 구분한다.
- 교체 가능성이 있는 항목은 한 곳에서 수정 가능하도록 상수화한다.
- 문서 원본을 직접 노출하기보다, 미리보기 카드와 보기 동작으로 웹에 맞게 구성한다.

## 구현 순서
1. 깨진 텍스트 데이터 정리
2. 상수/번역 구조 재정비
3. 헤더/푸터/공통 섹션 리디자인
4. 홈 페이지 재구성
5. 회사소개/기술력/인증현황/친환경/문의하기 순으로 재구성
6. 실자료 연결 정리
7. 로컬 검증

## 검증 기준
- `npm run lint`
- `npm run build`
- `npm run start` 기준 주요 페이지 응답 확인
- 최소 확인 경로
  - `/ko`
  - `/ko/about`
  - `/ko/technology`
  - `/ko/certification`
  - `/ko/sustainability`
  - `/ko/contact`
